# WealthOS Record Library v2

## Purpose

The Record Library helps users retrieve and correct financial history using the details they naturally remember.

## Primary interaction

One search field.

No Search button is required.

Search updates as the user types.

## Searchable information

- merchant
- income source
- account
- bill
- debt
- Roadmap
- category
- note
- amount
- date
- month
- source
- recurrence
- state
- record type

## Interaction hierarchy

1. Search
2. Quick filters
3. Optional advanced filters
4. Results
5. Data quality
6. Upcoming expectations

## Quick filters

- All time
- This month
- Last 30 days
- Recurring

## Advanced filters

- Type
- Recurrence
- Source
- Date from
- Date to

Advanced filters should never be required for normal retrieval.

## Search behavior

Search ranking favors:

- exact title match
- prefix match
- title match
- subtitle match
- metadata match
- amount match

## Trust and clarity

The UI explicitly states:

> Start typing. WealthOS searches as you type—no button required.

## Product principle

**People remember financial things—not database fields.**
